sol::object find_equivalent_key ( sol::table  t,

)

Find key of equivalent value in provided table.

Since userdata and tables are compared by their reference when used as keys, if we want to find value with a different userdata instance we first need to find a key that is equal by value to the one we want to use.

Parameters
ttable to search in for the key
desired_keyfound key will be equal to this value
Returns
sol::nil on failure, key object from table t on success
